# Browser Login Troubleshooting Guide

## Overview

This article helps support teams resolve browser-related sign-in, password reset, loading, redirect, and verification page issues. It follows common public browser help guidance: clear cached site data, confirm cookies are enabled, try a supported browser, and isolate extension conflicts.

## Key Concepts

- Cached files and cookies can preserve stale sessions, failed redirects, or old sign-in state.
- Some browser extensions can block scripts, cookies, pop-ups, redirects, or embedded sign-in pages.
- A private or incognito window can help isolate cache and extension behavior.
- Closing and reopening the browser after clearing data helps reset the session.
- If the issue persists across browsers or devices, support should collect details and escalate.

## Common Questions

### Why clear cache and cookies?

Clearing cache and cookies removes stale local browser data that may interfere with login pages, password reset pages, account verification, and loading screens.

### Where is the guide to clear cookies when nothing is loading?

If the interface does not load, ask the user to clear browser cache and cookies, close the browser completely, reopen it, and retry the sign-in or password reset page.

### What if extensions are blocking the page?

Ask the user to temporarily disable privacy, script-blocking, ad-blocking, or security extensions, or retry in a private window.

## Troubleshooting

1. Clear browser cache and cookies for the affected site or browser.
2. Close the browser completely.
3. Reopen the browser and retry the sign-in or password reset page.
4. Confirm cookies are enabled.
5. Try a supported browser or private window.
6. Temporarily disable extensions that may block redirects or scripts.
7. Capture browser name, visible error message, approximate time, and page URL if escalation is needed.

## Best Practices

- Give frustrated users short step-by-step instructions.
- Do not ask for passwords or verification codes.
- Avoid unnecessary technical jargon.
- Escalate if account recovery channels are unavailable or account changes are requested.
